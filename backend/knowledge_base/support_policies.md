# Base de conocimiento — Soporte

## Equipos y triage

| Equipo | Cuándo escalar |
|--------|----------------|
| **Soporte Técnico** | Fallos de producto, errores, configuración, integraciones. |
| **Facturación** | Cobros duplicados, reembolsos, facturas incorrectas, impuestos. |
| **Logística / Envíos** | Retrasos, pedidos no entregados, direcciones, devoluciones físicas. |
| **Cuentas y acceso** | Login, permisos, bloqueos de cuenta, restablecimiento de contraseña. |
| **Ventas / Pre-venta** | Demos, planes, upgrades antes de la compra. |

## Prioridad (orientativa)

- **critical**: servicio caído para muchos clientes o pérdida de datos.
- **high**: bloqueo total para un cliente o incumplimiento de SLA crítico.
- **medium**: impacto parcial o workaround disponible.
- **low**: consultas generales o mejoras cosméticas.

## SLAs (ejemplo)

- Respuesta primera interacción: objetivo 4 h laborables en **high/critical**.
- Resolución: depende del tipo; escalar a ingeniería si hay bug confirmado.

## Tono con el cliente

- Reconocer el inconveniente antes de pedir datos técnicos.
- No prometer plazos exactos sin verificar con el equipo responsable.
